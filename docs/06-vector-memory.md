# eye1.ai — Vector Database / Semantic Memory

## Ce se embed-uiește

| Artifact Type | Priority | Chunking | MVP |
|--------------|----------|----------|-----|
| Notion pages | HIGH | By heading sections | ✅ |
| GitHub PR descriptions + discussions | HIGH | Per PR (body + comments) | ✅ |
| GitHub Issues + discussions | HIGH | Per issue (body + comments) | ✅ |
| Conversation transcripts (Claude/ChatGPT) | HIGH | Per conversation turn pair | Phase 2 |
| WhatsApp threads | MEDIUM | Per thread (daily grouping) | Phase 2 |

## Chunking Strategy

### Default Strategy: Recursive Text Splitter
- **Chunk size**: 512 tokens (target)
- **Overlap**: 64 tokens
- **Splitting hierarchy**: heading → paragraph → sentence → token
- **Minimum chunk**: 50 tokens (discard shorter)

### Source-specific Strategies
- **Notion pages**: Split by H1/H2 headings, then by paragraphs
- **Conversations**: Split by turn pairs (user + assistant), keep context window of 2 previous turns
- **GitHub PRs**: Body as one chunk, each review comment as separate chunk
- **Code**: Not embedded (only metadata and descriptions)

## Metadata per Chunk

```typescript
interface ChunkMetadata {
  sourceType: string;          // "github", "notion", etc.
  sourceEntityType: string;    // "pull_request", "page", etc.
  sourceEntityId: string;      // External ID
  entityId: string;            // System UUID
  chunkIndex: number;          // Position in document
  totalChunks: number;         // Total chunks for this entity
  createdAt: string;           // ISO date of source creation
  updatedAt: string;           // ISO date of last update
  author?: string;             // Author name if available
  tags?: string[];             // Tags/labels if available
  language?: string;           // Detected language
  contentHash: string;         // SHA-256 of chunk content
}
```

## Re-embedding Strategy

- **Trigger**: Content update detected during normalization
- **Check**: Compare `contentHash` of new chunks vs existing
- **If changed**: Delete old embeddings, create new
- **Model change**: Batch re-embed all, tracked by `model` field
- **Schedule**: No proactive re-embedding unless model changes

## Embedding Versioning

- `model` field pe fiecare `EmbeddingRecord`
- La model switch, re-embed batch job (background, low priority)
- Old embeddings kept until new batch complete, then swapped atomically

## Retrieval

### Semantic Search
```typescript
async function semanticSearch(query: string, options: SearchOptions): Promise<SearchResult[]> {
  const queryEmbedding = await embed(query);
  return db.execute(sql`
    SELECT *, 1 - (embedding <=> ${queryEmbedding}) as similarity
    FROM embedding_records
    WHERE similarity > ${options.minSimilarity ?? 0.7}
    ${options.sourceType ? sql`AND metadata->>'sourceType' = ${options.sourceType}` : sql``}
    ${options.after ? sql`AND (metadata->>'createdAt')::timestamp > ${options.after}` : sql``}
    ORDER BY similarity DESC
    LIMIT ${options.limit ?? 10}
  `);
}
```

### Hybrid Search (Semantic + Full-text)
```typescript
async function hybridSearch(query: string, options: SearchOptions): Promise<SearchResult[]> {
  const semanticResults = await semanticSearch(query, options);
  const textResults = await fullTextSearch(query, options);
  return rerank(mergeResults(semanticResults, textResults), query);
}
```

### Recency-Aware Search
- Similarity score adjustat cu decay factor: `final_score = similarity * recency_boost`
- `recency_boost = 1.0 - (days_since_update / max_days) * decay_weight`
- `decay_weight = 0.3` (default) — recent content gets up to 30% boost

### Source-Aware Filtering
- Pre-filter by `sourceType` when user specifies source
- Post-filter by `sourceType` for diversity (ensure results from multiple sources)

## Prompt Injection Prevention

1. **Content sanitization**: Strip potential instruction patterns from imported content before embedding
2. **Metadata separation**: Chunk content is stored separately from system prompts — never concatenated directly
3. **Retrieval sandboxing**: Retrieved content is wrapped in clear delimiters:
   ```
   <retrieved_context source="notion" entity="page-123">
   {content}
   </retrieved_context>
   ```
4. **Confidence scoring**: Low-confidence retrievals are flagged
5. **Source trust levels**: System-generated content > verified sources > imported content

## Embedding Pipeline

```
[Canonical Write Event]
  → Check if entity type is embeddable
  → Extract text content
  → Chunk with source-specific strategy
  → Compute content hashes
  → Compare with existing embeddings
  → Embed changed/new chunks (batch API call)
  → Write EmbeddingRecords with metadata
  → Delete orphaned embeddings
  → Log audit (embed count, cost)
```

## Reranking Strategy

- **MVP**: Simple score fusion: `0.7 * semantic_score + 0.3 * text_score`
- **Phase 2**: Cross-encoder reranking model (Cohere rerank sau custom)
- **Phase 3**: User feedback-based reranking

## Quality Evaluation

- **Offline**: Manual test queries cu expected results
- **Online**: Track click-through on search results
- **Metrics**: Precision@5, Recall@10, MRR
- **Frequency**: Monthly evaluation pe set de 50+ test queries
